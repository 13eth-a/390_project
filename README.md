# **STAT 390 Capstone — Week 2 Baseline Pipeline**

**Modeling and Optimizing Political Ad Messaging Effectiveness**

---

## **Project Overview**

This project builds a reproducible baseline pipeline to predict political ad effectiveness using textual features from Facebook political advertisements. The goal of Week 2 is to establish a **fixed evaluation framework and baseline model** before introducing any automated experimentation.

Ad effectiveness is proxied using the **midpoint of impression ranges**, and the task is framed as predicting (and ranking) ads by performance.

---

## **Dataset**

### **Current Dataset (Week 2 Baseline)**

* Facebook political ads dataset: `fbpac-ads-en-US.csv`
* Expected at:

```bash
data/raw/fbpac-ads-en-US.csv
```

> **Note:** Large data files are not tracked in this repository. Download `fbpac-ads-en-US.csv` from [ProPublica's Facebook Political Ad Collector dataset](https://github.com/propublica/facebook-political-ads) and place it at the path above before running the pipeline.

### **Important Note**

This dataset was last updated several years ago and is being used **only to validate the modeling pipeline and evaluation framework** for Week 2.

### **Planned Dataset Upgrade**

In later weeks, this dataset will be replaced with a **recent export from the Meta Ad Library (2022–2024)** to align the analysis with modern political advertising trends.

### **Features Used**

* Ad text
* Sponsor (page name)
* Geographic targeting (if available)
* Impressions range (converted to midpoint)

---

## **Pipeline Overview**

The baseline pipeline follows a fixed, deterministic process:

```bash
Load data → Clean text → Extract features → Train model → Evaluate
```

---

## **Preprocessing**

* Lowercase text
* Remove URLs
* Normalize whitespace
* Drop missing or very short text
* Convert impressions range → midpoint
* Apply log transform: `log(1 + impressions)`

---

## **Baseline Model**

* **Features:** TF-IDF (max_features=5000, unigrams + bigrams)
* **Model:** Ridge Regression (`alpha=1.0`)
* **Target:** log-transformed impressions midpoint

This model is intentionally simple and serves as a reproducible benchmark.

---

## **Evaluation (FROZEN)**

### **Primary Metric**

* Spearman Rank Correlation (on validation set)

### **Secondary Metric**

* RMSE (on log-transformed impressions)

### **Output Format**

```json
{
  "spearman": <float>,
  "rmse": <float>,
  "runtime_seconds": <float>
}
```

⚠️ These metrics and format are fixed after Week 2.

---

## **Data Splitting (DETERMINISTIC)**

* Train: 70%
* Validation: 15%
* Test: 15%

Splits are:

* deterministic (hash-based)
* reproducible across runs

### **Important Rule**

* The test set is never used during development
* It will only be used for final evaluation
